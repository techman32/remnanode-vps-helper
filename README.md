# server-setup.sh

Интерактивный bash-скрипт для быстрого хардения нового VPS: управление портами через ufw, смена SSH-порта, блокировка ping, отключение входа по паролю, выпуск и управление SSL-сертификатами.

> **Примечание.** Скрипт написан с помощью нейросети (Claude). Каждый пункт меню протестирован и отвалидирован вручную. Основой для команд послужили личные наработки автора, накопленные за всё время администрирования серверов.

---

## Установка

Скачай, сделай исполняемым и запусти одной командой:

```bash
curl -fsSL https://raw.githubusercontent.com/techman32/remnanode-vps-helper/main/server-setup.sh -o server-setup.sh && chmod +x server-setup.sh && sudo ./server-setup.sh
```

Или по шагам:

```bash
# Скачать
curl -fsSL https://raw.githubusercontent.com/techman32/remnanode-vps-helper/main/server-setup.sh -o server-setup.sh

# Сделать исполняемым
chmod +x server-setup.sh

# Запустить (требуются права root)
sudo ./server-setup.sh
```

После запуска появится интерактивное меню. Выбирай нужный пункт цифрой и следуй подсказкам.

## Как работает каждый пункт

### 1. Открыть порты

Запрашивает протокол и список портов через запятую. По умолчанию — `80,443`.

1. Выбор протокола: `1` (TCP), `2` (UDP), `3` (оба)
2. Ввод портов через запятую

Для каждого порта проверяет что значение числовое и в диапазоне 1–65535, затем выполняет `ufw allow <порт>/<протокол>`. После обработки всего списка делает `ufw reload`.

Если ufw не установлен — устанавливает автоматически. Если выключен — включает.

---

### 2. Открыть порты для IP

Открывает порты только для конкретного IP-адреса. Трафик с других адресов на эти порты остаётся закрытым.

Запрашивает последовательно:

- **IP-адрес** — одиночный (`1.2.3.4`) или CIDR (`1.2.3.4/32`)
- **Протокол** — `1` (TCP), `2` (UDP), `3` (оба)
- **Порты** — через запятую, по умолчанию `80,443`

Создаёт правила вида `ufw allow from <ip> to any port <порт> proto <протокол>`.

---

### 3. Закрыть порты по IP

Удаляет IP-специфичные правила ufw, созданные через пункт 2.

Запрашивает последовательно:

- **IP-адрес**
- **Протокол**
- **Порты** — через запятую, по умолчанию `80,443`

Если правило не найдено — пропускает без ошибки.

---

### 4. Сменить SSH-порт

1. Читает текущий порт из `/etc/ssh/sshd_config`
2. Запрашивает новый порт
3. Заменяет строку `Port` в конфиге через `sed` (обрабатывает как закомментированную `#Port 22`, так и активную)
4. Открывает новый порт в ufw: `ufw allow <новый>/tcp`
5. Удаляет старый порт из ufw: `ufw delete allow <старый>/tcp`
6. Выполняет `systemctl daemon-reload` — без этого systemd не подхватывает изменения
7. Перезапускает SSH: пробует `ssh.socket` (Ubuntu 22.04+) и `sshd` одновременно
8. Проверяет что хотя бы один SSH-сервис активен

> ⚠️ Не закрывай текущую сессию до проверки подключения на новом порту.

---

### 5. Заблокировать ICMP (ping)

1. Проверяет наличие файла `/etc/ufw/before.rules`
2. Проверяет что блокировка ещё не настроена (идемпотентность)
3. Через `sed` меняет `-j ACCEPT` на `-j DROP` во всех ICMP-строках секций `ufw-before-input` и `ufw-before-forward`
4. Добавляет правило для `source-quench` (ufw не включает его по умолчанию)
5. Перезагружает ufw через `ufw disable && ufw --force enable` — `ufw reload` не применяет изменения в `before.rules`

После этого сервер перестаёт отвечать на ping.

---

### 6. Закрыть вход по паролю

Перед выполнением показывает предупреждение и требует ввести `yes` — защита от случайного локаута.

Устанавливает в `/etc/ssh/sshd_config` четыре директивы:

| Директива                         | Значение            | Что делает                                     |
| --------------------------------- | ------------------- | ---------------------------------------------- |
| `PasswordAuthentication`          | `no`                | Запрещает вход по паролю                       |
| `ChallengeResponseAuthentication` | `no`                | Отключает интерактивную аутентификацию         |
| `UsePAM`                          | `no`                | Отключает PAM (может разрешать пароль в обход) |
| `PermitRootLogin`                 | `prohibit-password` | Root может войти только по ключу               |

Каждая директива обрабатывается через `sed` — если строка закомментирована, раскомментирует и заменяет значение; если отсутствует — добавляет в конец файла. После этого перезапускает SSH.

> ⚠️ Убедись что SSH-ключ добавлен и ты можешь войти по нему, прежде чем запускать этот пункт.

### 7. Настройка SSL

Подменю для управления SSL-сертификатами Let's Encrypt и nginx.

```
── Настройка SSL ──────────────────────────
  1. Выпустить SSL-сертификат
  2. Сменить HTTPS-порт
  3. Сбросить SSL
  0. Назад
```

#### 7.1. Выпустить SSL-сертификат

Выпускает бесплатный TLS-сертификат Let's Encrypt и настраивает nginx.

Запрашивает:

- **Домен** — например `node.mydomain.com`
- **Email** — для уведомлений Let's Encrypt об истечении сертификата
- **Порт** — на котором nginx будет принимать HTTPS-трафик (по умолчанию `8443`)

Последовательность действий:

1. Устанавливает `nginx`, `certbot`, `python3-certbot-nginx` если отсутствуют
2. Открывает порт `80/tcp` в ufw — нужен certbot для HTTP-01 challenge и для каждого автопродления
3. Проверяет DNS: резолвит домен и сравнивает с внешним IP сервера — защита от расхода rate limit Let's Encrypt
4. Запускает `certbot certonly --nginx -d <домен>` — получает сертификат без изменения nginx-конфигов
5. Создаёт чистый файл `/etc/nginx/sites-available/<домен>` с SSL-конфигом только на указанный порт
6. Открывает указанный порт в ufw
7. Проверяет конфиг (`nginx -t`) и перезагружает nginx

После успешного выполнения домен доступен по `https://<домен>:<порт>`.

**Автопродление** работает через systemd timer (`certbot.timer`). Сертификат действует 90 дней, обновляется автоматически за 30 дней до истечения.

> Порт `80` должен оставаться открытым постоянно — он нужен при каждом автопродлении.

---

#### 7.2. Сменить HTTPS-порт

Меняет порт в nginx-конфиге для уже выпущенного сертификата без повторного обращения к certbot.

1. Спрашивает домен и показывает текущий порт
2. Запрашивает новый порт
3. Обновляет `listen` в `/etc/nginx/sites-available/<домен>`
4. Открывает новый порт в ufw
5. Предлагает закрыть старый порт в ufw
6. Перезагружает nginx

---

#### 7.3. Сбросить SSL

Полностью удаляет настройки SSL для домена — после этого можно выпустить сертификат заново через пункт 7.1.

1. Удаляет `/etc/nginx/sites-enabled/<домен>` и `/etc/nginx/sites-available/<домен>`
2. Перезагружает nginx
3. Предлагает удалить сертификат Let's Encrypt (`certbot delete`)
4. Предлагает закрыть HTTPS-порт в ufw

## Требования к системе

**Рекомендуется:**
- Ubuntu 22.04 LTS / 24.04 LTS
- Debian 12

**Совместимо (не тестировалось):**
- Ubuntu 20.04 LTS
- Debian 11

**Не поддерживается:**
- Ubuntu 18.04 и старше
- Debian 10 и старше

**Права:** root (`sudo`)

**Зависимости:**

| Пакет | Когда нужен | Установка |
|---|---|---|
| `ufw` | Все пункты с портами | Автоматически |
| `curl` | Пункт 7 (проверка DNS) | Автоматически |
| `dnsutils` (`dig`) | Пункт 7 (проверка DNS) | Автоматически |
| `nginx` | Пункт 7 (SSL) | Автоматически |
| `certbot`, `python3-certbot-nginx` | Пункт 7 (SSL) | Автоматически |
| `curl` или `wget` | Скачивание скрипта | Вручную |
